D. Detection engineering requirements
Sigma is an open, structured detection rule format designed so security teams can write shareable SIEM detections. Sigma rules are YAML files used to detect suspicious or malicious behavior in logs. 
MITRE ATT&CK is widely used as a knowledge base of real-world adversary tactics and techniques, and MITRE also provides guidance for using ATT&CK in detection analytics. 
Example:
title: Multiple Failed Logins From Same IP
logsource:
  product: windows
  service: security
detection:
  selection:
    EventID: 4625
  condition: selection
threshold:
  count: 5
  timeframe: 10m
level: medium
tags:
  - attack.t1110

8. AI features to build first
Start with these 5 because they will make your SIEM feel modern immediately:
AI-01: Alert explanation
Input:
{
  "rule": "Multiple Failed Logins",
  "events": [
    {"user": "admin", "src_ip": "10.0.0.55", "event_id": 4625}
  ]
}
Output:
This alert indicates repeated failed login attempts against the admin account from 10.0.0.55. This may represent brute-force activity or password spraying. Recommended next steps: check whether the source IP belongs to a known internal host, review successful logins after the failures, and verify whether the account was locked.
AI-02: Natural language query
User asks:
Show failed logins from the last 1 hour grouped by source IP.
System generates:
SELECT src_ip, count(*) 
FROM events 
WHERE event_id = 4625 
AND timestamp > now() - interval '1 hour'
GROUP BY src_ip
ORDER BY count(*) DESC;
AI-03: Incident timeline
AI builds:
10:02 - Multiple failed logins for admin
10:05 - Successful login for admin
10:07 - PowerShell executed encoded command
10:10 - New service created
10:13 - External connection to suspicious IP
AI-04: False-positive recommendation
This alert triggered 42 times in 24 hours from the vulnerability scanner host. Consider adding asset role = vulnerability_scanner as an exception, but keep detection active for all normal workstations.
AI-05: Report generator
Generate SOC incident report from selected alerts.
Output:
