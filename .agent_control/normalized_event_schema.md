# Normalized Event Schema

## Required Common Fields

| Field | Description |
|---|---|
| `event.id` | Stable event identifier. |
| `event.ingested_at` | Platform ingestion timestamp. |
| `event.created_at` | Source event timestamp. |
| `event.kind` | event, alert, metric, state, audit. |
| `event.category` | authentication, network, process, file, cloud, web, database, endpoint. |
| `event.type` | start, end, denied, allowed, failure, success, change, info. |
| `event.action` | Source action normalized to a comparable value. |
| `event.outcome` | success, failure, unknown. |
| `tenant.id` | Tenant identifier from auth/source context. |
| `source.name` | Source integration or agent name. |
| `source.type` | windows, linux, cloudtrail, firewall, proxy, dns, etc. |
| `host.name` | Hostname when available. |
| `user.name` | Principal username when available. |
| `source.ip` | Source IP address. |
| `destination.ip` | Destination IP address. |
| `process.name` | Process name when available. |
| `file.path` | File path when available. |
| `url.full` | URL when available. |
| `cloud.provider` | aws, azure, gcp. |
| `cloud.account.id` | Account/subscription/project identifier. |
| `rule.id` | Detection rule identifier if generated by detection. |
| `raw.ref` | Reference to raw source payload. |

## Schema Rules

- Preserve unknown source fields under `labels` or `source.fields`.
- Never discard raw evidence unless retention policy requires deletion.
- Normalize timestamps to UTC.
- Normalize severity to info, low, medium, high, critical.
- Normalize network actions to allowed, denied, blocked, dropped, unknown.
